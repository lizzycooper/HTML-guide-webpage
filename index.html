Creating the project

A "project" is a collection of files written in various programming languages that work together to create a webpage or application.

Each file will have a file extension that denotes the language it is written in. For an HTML/CSS project, you will need at least one document with the extension ".html", and one with the extension ".css". Here is an example of two typically named documents, in the Visual Studio Code application:

Each HTML document should have a <html> tag encompassing the entire document. This exists so that web browsers know how to interpret the code and display the page. Ensure the document has <html> in a line at the start, and </html> in a line at the end.

Don't know what a "tag" is?

HTML document head

The HTML document head contains important information, that doesn't necessarily render to the webpage the same way that the rest of the content does. Examples below will make this more clear.

Just below the initial <html> tag, insert an opening and closing tag like <head> </head>. All information in the head should be contained between these tags.

2. Linking to a stylesheet

The CSS document you created in the above section will be the stylesheet for the HTML document you create. Without a linked stylesheet, the browser will render the HTML as simply as possible, all in black Times New Roman.

Link to the CSS stylesheet as follows in the header section. If you have named your stylesheet something other than "styles.css", you will need to substitute the name.

'Link' is a self-closing tag. It contains some attributes inside it, and does not need extra information between an opening and closing tag.

Also note that you do not have to write your own stylesheet from scrach! External stylesheets can be downloaded, giving you default displays for each type of input.

HTML tags

Here are some basic HTML tags that you can use to create a webpage.

HTML tag Name and definition
<p>TEXT GOES HERE</p> Paragraph: renders the text to the webpage in a small font.
<em>TEXT GOES HERE</em> Emphasis: italicises or bolds a part of the text.
<h1>TEXT GOES HERE</h1> Heading: the largest heading is h1, and h2 is slightly smaller, etc. H6 is the smallest heading.
<figure>
      <img src="LINK-TO-IMAGE.jpg" title="TITLE HERE">
      <figcaption>CAPTION HERE</figcaption>
</figure>

There are a number of tags and attributes to define here.		 Figure: an inserted object (an image is just one example) that may have a caption or other attributes.
Image: the linked image will render on the page. Images can be stored locally (in the project folder), or can be linked from external sources (e.g. Google).
Title: If the browser has a problem displaying the image, the title will display instead. Visually impaired people can also read the title. Figcaption: A caption that will display on the page near the image (or other figure).
